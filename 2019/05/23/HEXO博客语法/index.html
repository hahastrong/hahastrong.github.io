HEXO博客语法

01标题

在markdown语法中，只需要在文本前面加 #即可，同理你还可以增加二级、三级、四级、五级、六级。总共有六级标题。例如：

    1. # 一级标题
    1. ## 二级标题
      < !–more–>
    1. ### 三级标题
    1. #### 四级标题
    1. ##### 五级标题
    1. ###### 六级标题

效果如下所示：

二级标题

02列表

在markdown中，添加列表也很简单。只需在文字前面添加 - 即可（- 和文本之间需要一个空格）。例如：

  • - 文本1
  • - 文本2
  • - 文本3

效果如下所示：

  • 文本1
  • 文本2
  • 文本3

03插入代码

在代码前使用四个空格或者一个制表符（tap）或者用反引号包裹住代码。
例如：

print "hello world"

注：三引号要与前面的文字内容空一行，不然代码不会要高亮显示，我在这里踩了个坑。

04链接和图片

绝对路径

当Hexo项目中只用到少量图片时，可以将图片统一放在source/images文件夹中，通过markdown语法访问它们。

![](/images/image.jpg)

图片既可以在首页内容中访问到，也可以在文章正文中访问到。


相对路径

图片除了可以放在统一的images文件夹中，还可以放在文章自己的目录中。文章的目录可以通过配置_config.yml来生成

_config.yml
1. post_asset_folder: true

将图片资源放在post_name中，文章就可以使用相对路径引用图片资源了。

_posts/post_name/image.jpg
1 ![](image.jpg)

上述是markdown的引用方式，图片只能在文章中显示，但无法在首页中正常显示。

如果希望图片在文章和首页中同时显示，可以使用标签插件语法。

_posts/post_name/image.jpg
1.  {% asset_img image.jpg This is an image %}



05插入音乐

比如网易云音乐，找到喜欢的歌曲，点击分享按钮，把里面的代码复制下来，直接粘贴到博文中即可。
比如插入以下代码:

<iframe frameborder="no" border="0" marginwidth="0" marginheight="0" width=330 height=86
</iframe>

06引用

在我们写作的时候经常需要引用他人的文字，这时引用格式就显得十分必要。在Markdown中，你只需要在你引用的文字前添加 > 即可，例如：

> 一盏灯， 一片昏黄； 一简书， 一杯淡茶。 守着那一份淡定， 品读属于自己的寂寞。 保持淡定， 才能欣赏到最美丽的风景！ 保持淡定， 人生从此不再寂寞。

最终显示的是；

一盏灯， 一片昏黄； 一简书， 一杯淡茶。 守着那一份淡定， 品读属于自己的寂寞。 保持淡定， 才能欣赏到最美丽的风景！ 保持淡定， 人生从此不再寂寞。

07粗体和黑体

Markdown 的粗体和斜体也非常简单，用两个 包含一段文本就是粗体的语法，用一个 包含一段文本就是斜体的语法。例如：

*一盏灯*， 一片昏黄；**一简书**， 一杯淡茶。 守着那一份淡定， 品读属于自己的寂寞。 保持淡定， 才能欣赏到最美丽的风景！ 保持淡定， 人生从此不再寂寞。

最终显示的文字如下：
一盏灯， 一片昏黄；一简书， 一杯淡茶。 守着那一份淡定， 品读属于自己的寂寞。 保持淡定， 才能欣赏到最美丽的风景！ 保持淡定， 人生从此不再寂寞。
