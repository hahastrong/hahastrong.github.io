20190608-C-RPN

Siamese Cascaded Region Proposal Networks for Real-Time Visual Tracking 论文地址

算法思路

通过叠加多个RPN结构以实现对不同卷积层特征进行融合与refine，有效地提升跟踪精度，减少网络对相似目标的误判。
整个网络的框架图如下图所示：


{ % asset_img 0608-1.png This is an image %}

上图三个RPN即为级联的RPN模块，用于多特征融合以及refine目标坐标。

motivation

  • 目前训练样本的分布不均，负样本数量过多，并且负样本过于简单
  • 底层特征信息未被使用， 高层特征难以区分相似目标
  • two-stage可以获得更加精准的位置信息即使目标发生较大的形变

贡献

  • 提出多stage跟踪框架，用于解决类不平衡问题
  • 与one-stage SiamRPN相比，在位置定位上多stage更加准确，不同层间的特征使用提升了模型的分辨力

做法

级联多个RPN结构，低级RPN将结果送给下一级RPN，并传递anchor信息，提出简易负样本。
越低级的RPB使用的特征越高层。
不同层间的特征融合使用作者提出的FTB（Feature Transfer Block）模块。模块具体结构如下图


{ % asset_img 0608-2.png This is an image %}

不同层特征通过解卷积、卷积、差值得到。

训练过程中，网络的负样本是最后一级RPN中的负样本，因为训练的负样本数量大大减少，达到正负样本比例均衡。


{ % asset_img 0608-3.png This is an image %}

上图为三个不同级别RPN的响应图，从左往右级别逐渐升高，可以看出高级别的RPN已经有效地提出那些相似样本。使得结果更加准确。

性能


{ % asset_img 0608-4.png This is an image %}

上图为CRPN与其他算法在OTB2013和OTB2015数据集上的AUC曲线图。CRPN相对于SiamRPN分别有1.7%和2.6%的性能提升。证明多层特征融合对性能提升是有帮助的。

Thoughts

  • 通过级联RPN减少easy负样本，达到正负样本保持平衡。同时融合多层特征，提升定位目标的精度。
  • 级联RPN对速度应该有部分的损失，3-stage只有23fps。有点得不偿失。

20190607-zongjie

期末考快到了，大作业也随之而来。
这次机器视觉的大作业很要命，看20篇文献写个综述。
还好之前看了几篇CV相关的文献，不然14天之内怎么能看得完加写好综述。。。。

列列我近期的打算：

  • 看20篇文献并写个4000字的综述
  • 刷LeetCode
  • 阅读原著30分钟
  • 每日整理一句长难句

  • 刷统计学习方法

  • 学习算法导论

  • 学习C++

以上这些计划有些事相辅相成的，比如再刷LeetCode的同时可以巩固c++语法知识，也会用到算法的知识，举一反三。哈哈哈哈哈哈

具体的时间该怎么划分呢？
具体问题得具体分析！！！
阅读与整理长难句属于日常操作，所以无需多加计划。只需每日提醒自己即可。（阅读花费时间： 30min，长难句花费时间：20min）
LeetCode 需每周刷两题，周一周四各一题。每周二周五需要将下一次LeetCode题打印下来，提前看看，平时想想解题思路。（每题花费时间：1h）
统计学习方法 有两个多月没看了。。。。这次暂且每周一章，周二学习做笔记，周三笔记做成博客形式。（每次花费时间：~2h）
算法导论 大块头！！！ 每周三周六看20页并做笔记即可。（每次花费时间：1h）

那20篇文献，每篇需要花费时间：2h，总共花费时间：40h。每天看两篇，还需看6天。每篇论文做笔记需30min。
每天固定花费时间大概是7h，一天有效学习也就7h。。。。其他时间就看看杂书吧。再刷刷微博。。溜溜星球。。。跑跑步吧！
